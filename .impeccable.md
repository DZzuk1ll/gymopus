# Impeccable Design Context

## Design Context

### Users
有经验的健身训练者。他们熟悉训练术语和动作模式，不需要手把手引导。使用场景包括训练前的计划制定和健身知识查询。他们重视效率，期望快速获取精准信息，而非浏览花哨的界面。

### Brand Personality
**专业 / 智能 / 活力**

- **语气**：直接、简练、有信心。像一个靠谱的训练搭档，不废话，给出有依据的建议。
- **情绪目标**：简洁高效——用户应该感到这个工具懂行、值得信赖、用起来快。
- **反面参考**：不要花哨装饰、不要游戏化元素、不要过度动画。

### Aesthetic Direction
- **视觉基调**：干净利落 + 克制的活力感。大量留白，信息密度适中，关键数据突出。
- **配色策略**：以中性色为主体，用一个饱和度适中的强调色注入活力（推荐方向：深青/靛蓝/琥珀等运动感色调）。避免多色混杂。
- **排版**：Geist 字体族保留。标题紧凑有力，正文清晰可读。数字和训练数据可用 Mono 字体强调。
- **主题**：同时支持亮色和暗色模式，均为一等公民。
- **图标**：用 Lucide 图标替换当前的 emoji 导航，保持一致的视觉语言。
- **动效**：仅在有功能意义时使用（状态转换、反馈），不做纯装饰动画。

### Design Principles

1. **效率至上 (Efficiency First)**
   每个元素必须有存在的理由。去掉一切不帮助用户完成目标的装饰。

2. **克制的力量感 (Confident Restraint)**
   足够大胆以传递活力，足够克制以保持专业。强调色少而精准。

3. **数据优先于装饰 (Data Over Decoration)**
   训练计划、组数、次数、休息时间——这些才是主角。用排版层级和间距让数据自己说话。

4. **智能默认 (Intelligent Defaults)**
   减少用户决策负担。界面应该感觉"它懂我"——合理的预设、聪明的布局、最少的点击。

5. **有目的的能量 (Purposeful Energy)**
   色彩和动效只在需要引导注意力或提供反馈时出现。静止时优雅，交互时有力。

## Technical Notes

- **框架**: Next.js 16 (App Router) + React 19 + TypeScript
- **样式**: TailwindCSS 4 + shadcn/ui (基于 @base-ui/react)
- **字体**: Geist (sans) + Geist Mono (mono)
- **色彩空间**: OKLch
- **图标库**: lucide-react
- **组件变体**: CVA (class-variance-authority)
- **状态管理**: TanStack Query + localStorage
- **国际化**: 中文优先 (zh-CN)
